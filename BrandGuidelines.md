# ColonySearch — Brand Guidelines

**Version 1.0 · April 2026**

---

## 1. Brand Essence

### 1.1 The Idea in One Line
> **No central index. No single owner. Queries find their own way.**

### 1.2 Brand Personality
ColonySearch is **technical but humane**, **principled but pragmatic**, **decentralized but coherent**. The brand should feel like infrastructure you can trust: clean, calm, confident — never loud, never corporate-flashy.

| We are | We are not |
|---|---|
| Open, transparent, sovereign | Anti-establishment, edgy, conspiratorial |
| Scientific, precise, evidence-led | Cold, academic, inaccessible |
| Bio-inspired, organic, emergent | Mystical, naturalistic, decorative |
| Minimalist, structured, modern | Sparse, sterile, generic-tech |

### 1.3 Voice & Tone
- **Short sentences. Strong claims.** ("One Index. One owner. Zero Transparency.")
- Use **periods as design elements** — they create rhythm and authority, as seen on the poster.
- Prefer concrete nouns over marketing adjectives ("index", "node", "query", "swarm" — not "synergy", "leverage").
- Technical when needed, plain when possible. Never both at once.

## 2. Logo & Mark

The primary mark pairs a **magnifying-glass icon** with a **distributed-node graph** inside it — search meets swarm. This combination is the visual lock of the brand and should remain intact wherever possible.

**Wordmark composition:** `Colony` + `Search`
- "Colony" is set in the primary text color (`#161b22`).
- "Search" is set in the brand blue (`#1f48ff`).
- Always one word, no space, two-tone.

**Clear space:** Minimum padding around the logo equal to the height of the lowercase "o" in "Colony".

**Don'ts:**
- Don't recolor "Colony" to blue or "Search" to black.
- Don't separate the mark from the wordmark in headers or covers.
- Don't place the logo on busy photographic backgrounds — use a solid panel.

## 3. Color System

### 3.1 Palette

| Role | Name | HEX | Usage |
|---|---|---|---|
| Primary text | **Ink** | `#161b22` | Body text, headings, primary UI text |
| Primary accent | **Swarm Blue** | `#1f48ff` | "Search" wordmark, links, primary CTAs, key highlights |
| Positive accent | **Colony Green** | `#00bf63` | Affirmations, success states, sustainability/SDG context |
| Energy accent | **Signal Orange** | `#ff751f` | Icons, secondary highlights, attention markers |
| Surface | **Mist** | `#e6edf3` | Cards, panels, subtle separation on white backgrounds |
| Base | **White** | `#ffffff` | Default canvas |

### 3.2 Color Logic (How the Poster Uses It — and How You Should Too)
- **Blue = the product.** Anything central to the search proposition (the word "Search", primary actions, key headings).
- **Green = the values.** Affirmative statements, the manifesto line, SDG and sustainability contexts.
- **Orange = the function.** Icons and feature markers (Global, Seed, Private Seed Search modes).
- **Ink = the substrate.** All running text and structure.
- **Mist = the breathing room.** Use as section panels — never as text color.

### 3.3 Proportional Use
Aim for roughly **70 % neutral** (white + Mist + Ink), **20 % Swarm Blue**, **10 % Green + Orange combined**. Accents earn their weight by being rare.

### 3.4 Accessibility
- Ink on white and Ink on Mist both meet WCAG AAA for body text.
- Swarm Blue on white passes AA for normal text — preferred for links and headings.
- Avoid Green or Orange for body copy; reserve them for short text, icons, and highlights.

## 4. Typography

### 4.1 Typefaces

| Use | Typeface | Weights |
|---|---|---|
| Headings, wordmark, poster titles | **DM Sans** | 500 / 700 |
| Body, captions, UI, technical text | **Inter** | 400 / 500 / 600 |

Both are open-source (Google Fonts / OFL) — a deliberate choice that mirrors the project's open-source philosophy.

### 4.2 Type Scale (Suggested)

| Level | Font | Size | Weight | Use |
|---|---|---|---|---|
| Display | DM Sans | 48–64 px | 700 | Poster titles, hero claims |
| H1 | DM Sans | 32 px | 700 | Document titles |
| H2 | DM Sans | 24 px | 700 | Section headers |
| H3 | DM Sans | 18 px | 500 | Subsections |
| Body | Inter | 16 px | 400 | Running text |
| Small / caption | Inter | 13 px | 500 | Labels, footnotes |
| Code / mono | (system mono) | 14 px | 400 | Tech stack, snippets |

### 4.3 Typographic Style
- **Headlines**: tight leading (1.1–1.2), no italics, sentence case or title case — never ALL CAPS for long phrases.
- **Body**: leading 1.5, max line length ~70 characters.
- **Emphasis**: prefer **bold (Inter 600)** over italics. Reserve italics for the manifesto line and direct quotes.
- **Punctuation as rhythm**: short, period-separated phrases ("Your data. Your boundaries. Complete data sovereignty.") are part of our voice — use them in headers and feature blocks.

## 5. Iconography & Imagery

### 5.1 Icons
- **Style:** flat, outlined, geometric — built on a node-and-edge logic where possible.
- **Color:** Signal Orange as default fill/stroke for feature icons (Global / Seed / Private), echoing the poster.
- **Stroke weight:** consistent across an icon set; don't mix outline and filled styles in the same view.

### 5.2 The Network Motif
The blue/green/teal node-graph illustration is a **brand-defining visual**. Use it — or stylized variants — to:
- anchor hero sections of slides, posters, and reports;
- represent the swarm conceptually wherever architecture is discussed.

Rules:
- Always render on white or very light backgrounds.
- Nodes use Swarm Blue and Colony Green (with neutral teal mid-tones).
- Edges are thin, light gray (`#c9d1d9` or similar) — never black.
- Avoid drop shadows, glow effects, or 3D treatments.

### 5.3 SDG Badges
SDG 10 and SDG 16 logos are **official UN assets** — never recolor, crop, or distort them. Place them on white with adequate clear space, as on the poster.

---

## 6. Layout Principles

1. **Grid first.** Use a clear column grid; align everything to it. The poster's three-column rhythm (problem · concept · modes) is the canonical pattern.
2. **Panels, not borders.** Group content with Mist (`#e6edf3`) panels and rounded corners — not heavy outlines.
3. **One idea per block.** Each card answers one question.
4. **Whitespace is a feature**, not wasted real estate.
5. **Hierarchy through size and weight**, not through color alone.

## 7. Application Examples

| Context | Recommended Treatment |
|---|---|
| **Slide deck (Accenture / jury)** | White background · DM Sans titles in Ink · single Swarm Blue accent per slide · network motif on title + section dividers |
| **Poster** | The reference application — keep this layout language consistent across future posters |
| **PDR / written reports** | White, generous margins, Inter body, DM Sans headings, Swarm Blue for links and key terms |
| **Code / GitHub** | README with the wordmark, network motif as social preview, monospace for tech stack |
| **UI prototype** | White canvas · Mist panels · Swarm Blue primary buttons · Orange for mode selectors · Green for confirmations |
